Getting Started with Composite Micro Framework

by David Echeverri

Composite: an html component loader that allows you to load web components with a simple line 
data-load="doc.html"

The problem we are addressing is the haunting need for large frameworks and libraries to simply embed HTML inside HTML without using iframes but in a more elegant way. Composite provides a lightweight solution for creating web components using the data-load attribute to load HTML snippets or chunks.

Full Page Skeleton Example

Below is an example of a full page skeleton or boilerplate with Composite integrated:

  <!DOCTYPE html>
  <html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Composite App</title>

    <style>
      /* Your custom styles can go here */
    </style>
  </head>
  <body>

    <h1>Welcome to My Composite App</h1>
    <div data-load="component1.html"></div>
    <div data-load="component2.html"></div>


    <script src="composite.js"></script>
    <script>
      document.addEventListener('DOMContentLoaded', loadComponents);
    </script>

  </body>
  </html>

How to Use Composite

To use Composite in your projects, follow these steps:

  1. Include the Composite script in your HTML file:
    2. <script src="composite.js"></script>
  2. Add the data-load attribute to your HTML elements to specify the path to the HTML snippet you want to load:
    3. <div data-load="component1.html"></div>
  3. Initialize Composite by calling the loadComponents function when the DOM content is loaded:
    4. <script>
    document.addEventListener('DOMContentLoaded', loadComponents);
  </script>

Made by a designer for designers and developers

Composite was crafted with the needs of both designers and developers in mind. Its simplicity streamlines the process of creating component-based prototypes, eliminating unnecessary complexity and allowing designers to focus on their creativity without getting bogged down by technical details. By providing an intuitive and straightforward framework, Composite empowers designers to efficiently express their ideas and iterate on designs, while also offering developers a seamless integration process.

Rapidly prototype without any build process or massive directories, keep it flat and elegant

With Composite, prototyping becomes a breeze, eliminating the need for cumbersome build processes or extensive directory structures. By keeping the workflow flat and elegant, designers can quickly modularize their work into smaller, more manageable chunks. This approach encourages a more iterative and agile design process, allowing designers to experiment with different components and layouts without being hindered by organizational overhead. By breaking down designs into smaller, more focused units, designers can maintain better control over their projects and adapt more easily to changes, resulting in more efficient and effective prototyping.

        function loadComponents() {
            const componentContainers = document.querySelectorAll('[data-load]');
            componentContainers.forEach(container => {
                const url = container.getAttribute('data-load');
                fetch(url)
                    .then(response => response.text())
                    .then(html => {
                        const shadowRoot = container.attachShadow({ mode: 'open' });
                        shadowRoot.innerHTML = html;
                    })
                    .catch(error => console.error('Error loading component:', error));
            });
        }

        document.addEventListener('DOMContentLoaded', loadComponents);
